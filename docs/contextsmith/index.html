Pipeline

From documents
to answers.

Six composable stages take your raw documents through loading, chunking, embedding, storage, retrieval, and generation — each fully replaceable with a custom adapter.

pipeline.txt
Documents (file / URL / text)
    │
    ▼
[ Loader ]       — reads text from any source
    │
    ▼
[ Chunker ]    — splits into overlapping windows
    │
    ▼
[ Embedder ]   — embeds each chunk (OpenAI / custom)
    │
    ▼
[ Vector Store ] — stores chunks (in-memory or custom)
    │  Query time:
    ▼
[ Retriever ]   — embed query → cosine search → top-k
    │
    ▼
[ Generator ]   — context + LLM prompt → answer + sources
📂
Loader
Reads text from files, URLs, or plain text strings.
✂️
Chunker
Sliding-window splits with configurable size & overlap.
🧠
Embedder
Vector representation via OpenAI or a custom adapter.
🗄️
Vector Store
In-memory by default; plug in Pinecone, pgvector, Qdrant.
🔍
Retriever
Cosine similarity search with optional reranking.
✨
Generator
Stuffs retrieved context into an LLM prompt and returns an answer with cited sources.

Sources

Index anything.
Mix and match.

Pass a single source type or combine all three in one index() call. The pipeline handles the rest.

type: "file"
File Upload
Pass a browser File object directly. PDF, plain text, markdown — anything with readable text content.
type: "url"
URL
Provide an HTTP/HTTPS URL. The loader fetches and extracts the page text automatically.
type: "text"
Raw Text
Paste any string directly. Add optional metadata like name to label it in citations.
index-sources.ts
const { index } = useRAG({ config })

// Mix and match sources in one call
index([
  { type: "file", content: resumeFile },
  { type: "url", content: "https://docs.example.com/api" },
  { type: "text", content: internalNote, metadata: { name: "My Note" } },
])

Providers

Mix LLM &
embedding providers.

Configure the LLM for generation and the embedding model independently. Swap either with a custom adapter at any time.

Anthropic

Default: claude-sonnet-5
OpenAI
llm.provider: "openai"
Default: gpt-4o
Mistral
llm.provider: "mistral"
Default: mistral-large-latest
Custom
llm.provider: "custom"

Hook API

One hook.
Full control.

Use useRAG to drive the pipeline from any React component. All indexing, querying, and state management in one place.

index(sources)
Promise<void>
Load, chunk, embed, and store one or more document sources. Tracks progress via indexingProgress.
query(question)
Promise<void>
Embed the question, retrieve top-k chunks, generate an answer. Result available in lastResult.
removeDocument(id)
Promise<void>
Delete a specific document and all its chunks from the vector store.
clearIndex()
Promise<void>
Wipe the entire index — all documents and embeddings.
Return Value Type Description
indexedDocuments RawDocument[] All documents currently in the index.
totalChunks number Total number of chunks stored across all documents.
lastResult QueryResult | null Answer, cited sources, and duration from the most recent query.
indexingProgress IndexingProgress Stage-by-stage progress indicator while indexing is running.
queryStatus QueryStatus Retrieval and generation status for the current query.
error Error | null Last error thrown by any pipeline stage.

Prompt templates

Context injected
automatically.

Use template variables in generation.systemPrompt. The pipeline fills them in before every LLM call.

Variable Description
{{context}} Retrieved chunks, formatted with source names and similarity scores.
{{query}} The user's verbatim question.

DocumentQA.tsx
import { ContextForge } from "contextsmith"

const config = {
  llm: {
    provider: "anthropic",
    apiKey: import.meta.env.VITE_ANTHROPIC_KEY,
    model: "claude-sonnet-5",
  },
  embeddings: {
    provider: "openai",
    apiKey: import.meta.env.VITE_OPENAI_KEY,
    model: "text-embedding-3-small",
  },
  retrieval: { topK: 5, scoreThreshold: 0.3 },
  generation: { citations: true },
}

export default function DocumentQA() {
  return <ContextForge config={config} placeholder="Ask about your docs…" />
}
