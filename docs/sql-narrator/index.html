RevenueByRegion.tsx
import { SqlNarrator } from "sql-narrator"
<SqlNarrator
  llm={{ provider: "anthropic", apiKey: KEY }}
  dialect="snowflake"
  tone="analyst"
  sql={`
    SELECT region, SUM(amount) AS revenue
    FROM sales WHERE order_date >= ...
    GROUP BY region ORDER BY revenue DESC
  `}
/>
SQL Narrator snowflake · analyst
Ranks total revenue by region for orders in the last three months.
What the query does
Aggregates amount from the sales table into a per-region revenue total, filtering to orders whose order_date falls within the last three months (Snowflake's DATEADD), then sorts regions from highest to lowest revenue.
What the results mean
EMEA leads at $1.24M — roughly 27% ahead of AMER ($980K) and nearly 3× APAC ($445K). The distribution suggests the top two regions are driving the vast majority of quarter revenue.
Caveats
Snowflake's DATEADD is inclusive of today — results include partial current-day data.

HeadlessHook.tsx
import { useSqlNarrator } from "sql-narrator"

const { narration, loading, error, narrate } = useSqlNarrator({
  provider: "openai",
  apiKey: process.env.NEXT_PUBLIC_OPENAI_KEY,
  model: "gpt-4o",
})

await narrate({ sql: myQuery, dialect: "postgres", tone: "teacher" })
// narration = { summary, queryExplanation, resultsInterpretation?, caveats }

Props

Small surface.
Composable core.

llm: LLMConfig
Required. provider, apiKey, and optionally model, baseUrl, headers.
sql: string
Required. The query to narrate.
results?: QueryResults
{ columns, rows, totalRowCount? }. When provided, enables results interpretation.
dialect?: SqlDialect
One of snowflake, postgres, mysql, sqlserver, bigquery, oracle, ansi. Defaults to ansi.
tone?: NarrationTone
narrative, analyst, casual, teacher. Defaults to analyst.
context?: string
Business context, e.g. "runs nightly against the sales mart".
maxSampleRows?: number
Cap on result rows sent to the LLM. Defaults to 20.
auto?: boolean
Re-narrate when sql or results change. Defaults to true.
