Pipeline

Agents that
think together.

Each agent has a role, a system prompt, and optionally its own LLM. They pass context to one another and a Synthesizer produces the final conclusion.

pipeline.txt
User input
    │
    ▼
┌──────────────────────────────┐
│          sequential pipeline         │
│ Agent 1 → Agent 2 → Agent 3 → … │
└──────────────────────────────┘
    │
    ▼
Synthesizer (sees all outputs via {{outputs}})
    │
    ▼
Final reasoning output

In parallel mode, all agents run simultaneously with the original input. The synthesizer still waits for all of them.

Orchestration

Sequential
or parallel.

Choose how agents relate to each other. One setting on the pipeline config.

➡️
Sequential
mode: "sequential"
Each agent receives the previous agent's output via {{previous}} and all prior context via {{context}}. Builds on cumulative reasoning — ideal when agents should react to each other.
⚡
Parallel
mode: "parallel"
All agents run simultaneously against the original input. Faster. Better when you want independent, uninfluenced perspectives from each agent.

Providers

Mix providers
in one pipeline.

Set a pipeline-level fallback and override it per agent. Use GPT-4o for speed, Claude Opus for depth, all in one run.

Anthropic
provider: "anthropic"
Default: claude-sonnet-4-20250514
OpenAI
provider: "openai"
Default: gpt-4o
Mistral
provider: "mistral"
Default: mistral-medium
Custom
provider: "custom"
Bring your own adapter — implement the LLMAdapter interface.

Prompt templates

Context flows
automatically.

Use template variables in any system prompt. No manual wiring.

Variable Available in Description
{{input}} All agents & synthesizer The original user input.
{{previous}} Sequential agents Output of the immediately preceding agent.
{{context}} Sequential agents All prior agent outputs, formatted chronologically.
{{outputs}} Synthesizer All agent outputs, formatted with agent names.

StrategyReview.tsx
import { MultiAgentReasoning } from "agentquorum"

const pipeline = {
  mode: "sequential",
  llm: { provider: "anthropic", apiKey: process.env.ANTHROPIC_KEY },
  agents: [
    { id: "analyst", role: "Break down the problem", icon: "📊",
      systemPrompt: "Analyze: {{input}}" },
    { id: "critic", role: "Challenge assumptions", icon: "⚡",
      systemPrompt: "Analyst said: {{previous}}. What's wrong with this?" },
  ],
  synthesizer: { systemPrompt: "All outputs:\n{{outputs}}\n\nFinal verdict:" },
}

<MultiAgentReasoning
  pipeline={pipeline}
  placeholder="What should the council reason about?"
/>

Hook API

Full control
with useMultiAgent.

Skip the component and drive the pipeline directly. Plug results into any UI.

run(input: string)
Start the pipeline with a user input string. Returns a Promise that resolves when synthesis is complete.
agentResults: AgentResult[]
Live-updating list of completed agent outputs. Updates as each agent finishes — great for streaming UIs.
synthesis: string
Final synthesizer output. Available once all agents have completed and the synthesizer has run.
activeAgentId: string | null
The ID of whichever agent is currently running. Use this to highlight the active step in your UI.
running: boolean
True while any part of the pipeline is active. Use for loading indicators.
reset()
Clear all agent results, synthesis, and error state. Resets the pipeline to its initial state.
