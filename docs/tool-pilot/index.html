ReAct Loop

Think. Act.
Observe. Repeat.

The agent follows a ReAct (Reason + Act) loop. It reasons about what to do, calls a tool, reads the result, and continues until it has a grounded answer.

Built-in Tools

Three tools.
Infinite tasks.

Ship with batteries included. Or create your own tools in minutes.

🔍
Web Search
webSearch
Search the web via DuckDuckGo Instant Answer API. Returns summaries and related topics. Override with your preferred search provider.
💻
Code Execution
codeExec
Execute JavaScript in a sandboxed environment. console.log output is captured and returned. Great for calculations, data transforms, and logic.
📄
File Read
fileRead
Fetch and read the text content of any URL. HTML tags are stripped for readability. Content is truncated to configurable max length.

Extensible

Build any
tool you need.

Implement ToolDefinition — name, description, parameters, and an execute function. The LLM sees the schema automatically.

weatherTool.ts
const weatherTool: ToolDefinition = {
  name: "get_weather",
  description: "Get current weather for a city",
  parameters: [
    { name: "city", type: "string", required: true },
  ],
  execute: async (args) => {
    const res = await fetch(`https://wttr.in/${args.city}?format=3`)
    return res.text()
  },
}

Reasoning Trace

Every step
is visible.

Watch the agent's full chain of thought. Each step is streamed to the UI in real time.

Kind Icon Description
thinking 🧠 LLM reasoning about what to do next
tool-call 🔧 Agent chose a tool and is calling it with arguments
observation 👁 Result returned from the tool execution
answer ✅ Final answer grounded in observations
error ❌ Error during LLM call or tool execution

Hook API

Full control
with useToolPilot.

Skip the component. Drive the agent from any UI or use it headless.

run(input: string)
Start the agent with a goal. Returns a Promise that resolves when the agent produces an answer.
steps: ReasoningStep[]
Live-updating list of reasoning steps. Streams each think/tool-call/observation as it happens.
answer: string
Final answer produced by the agent after reasoning through tools.
status: AgentStatus
"idle" · "planning" · "executing" · "done" · "error" — track the agent's state.
error: Error | null
Error object if the agent failed. null during normal operation.
reset()
Clear all steps, answer, and error state. Returns the agent to idle.
