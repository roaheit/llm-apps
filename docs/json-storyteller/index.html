Tones

Data has
a voice.

The same JSON can read like a business report, a casual summary, or a story. Pick the right voice for your context.

📊
Analyst
tone="analyst"
Data-driven, precise language. Numbers front and center. Ideal for dashboards and reports.
📖
Narrative
tone="narrative"
Flowing prose that tells the story behind the data. Best for user-facing summaries.
💬
Casual
tone="casual"
Conversational and approachable. Great for consumer apps and onboarding flows.

/>

Props

Three props.
That's the API.

llm: LLMConfig
Provider config object. Requires provider and apiKey. Optionally accepts model to override the default.
data: object
Any JSON-serializable object. Nested objects, arrays, numbers — all handled automatically.
tone: "narrative" | "analyst" | "casual"
Controls the style and voice of the generated narrative. Defaults to "narrative".
adapter?: LLMAdapter
Optional custom adapter for any provider not natively supported. Implement the LLMAdapter interface and pass it here.
