Architecture

Three layers.
One clean API.

Most agents have no memory. The rest have bad memory. agent-recall gives you all three types — properly.

⚡
Short-term
In-process buffer for the current session. Stores recent messages and observations. Auto-compresses to long-term when the token limit is hit.
In-session
🧠
Long-term
Cross-session semantic memory. Embeddings stored in a local vector DB. Retrieved via similarity search — only what's relevant, not everything.
Persistent
📋
Episodic
Timestamped log of agent actions and outcomes. Filter by time, tag, or action type. Lets your agent learn from what it's already done.
Queryable

quickstart.py
from agent_recall import Memory
import os

mem = Memory(agent_id="my-agent")

# Store a fact
mem.remember("User's name is Alex", layer="long_term")

# Retrieve relevant context before your LLM call
context = mem.recall("who is the user?")

# Log what your agent did
mem.log_episode(action="api_call", result="success")

# Compress buffer when it gets long
if mem.short_term.is_full():
    mem.compress()

API

Four methods.
That's it.

mem.remember(content, layer)
Store a memory in short_term or long_term. Optional metadata dict for tagging and filtering.
mem.recall(query, layers)
Retrieve relevant memories via semantic search. Returns formatted string ready to inject into a prompt.
mem.log_episode(action, result, tags)
Append a timestamped action/outcome to the episodic log. Filter later by tag, time range, or action name.
mem.compress()
LLM-summarizes the short-term buffer, moves the summary to long-term, and clears the buffer.
